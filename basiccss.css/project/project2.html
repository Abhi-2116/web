How does this work?

The key a lot of this is leveraging using tables with a attribute to make decorated sectios without css. if you view source on the page. you`ll see that there a lot of tables. the table tag typically is used to display tabular data and should NOT be used a layout method. However, since I am not allowed to use css, I am pretty much exclusively relying on tables for this purpose. the presentation role tells the browser to not eppose the table and its tablur children in the accessbility tree. this will make it easier to navigate for assitive technology.

I`m also leverging some deprected HTML elements from HTML 3.2 that you should never use. in order to be compliant,I`ve addes the HTML 3.2 doctype at the top of the page:

This helps give a hint to the browser that we are going to use some colour stuff.

HOWEVER I`m also using some never things such as details/summary,section,main,header,and footer. So I`m just making a mess of my doctype and what I should be actually using!
